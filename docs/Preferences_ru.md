# Настройки

## Обнаружение субтитров

<img src="images/sub%20det.png" width="400">

**Split Start (Начало отрезка)**: Позиция кадра, с которой начнется обнаружение субтитров. Значение относительно общей длины видео. 0.0 означает начало обнаружения с самого старта видео.

**Split Stop (Конец отрезка)**: Позиция кадра, на которой остановится обнаружение субтитров. Значение относительно общей длины видео. 0.5 остановит обнаружение на середине видео, а 0.95 — ближе к концу.

**No of Frames (Количество кадров)**: Количество кадров между начальной и конечной позициями, которые будут использованы для обнаружения субтитров. Слишком мало кадров — и программа не обнаружит субтитры. Слишком много — обнаружение займет больше времени и может захватить больше текста, не являющегося субтитрами, что некорректно увеличит ограничивающую рамку.

**X Axis Padding (Отступ по оси X)**: Определяет длину горизонтальной линии ограничивающей рамки. Алгоритм обнаружения субтитров не может быстро определить максимальную длину по оси X, просматривая все кадры. Значение 1.0 означает, что самая длинная строка субтитра занимает всю ширину видео. Значение относительно ширины видео.

**Y Axis Padding (Отступ по оси Y)**: Предотвращает слишком близкое расположение вертикальной линии ограничивающей рамки к обнаруженному тексту субтитров. Это значение фиксировано для всех высот видео.

**BBox Drop Score (Минимальный порог уверенности для рамки)**: Минимально допустимый показатель уверенности для каждой обнаруженной ограничивающей рамки. Низкое значение приведет к менее точному итоговому определению области субтитров, которая может включать участки, не содержащие текста субтитров.

**Use Default Search Area (Использовать область поиска по умолчанию)**: Для обнаружения субтитров будет просканирована область кадра, заданная по умолчанию. Если снять флажок, сканируется весь кадр видео.

## Извлечение кадров

<img src="images/frame%20extract.png" width="400">

**Frame Extraction Frequency (Частота извлечения кадров)**: Частота извлечения кадров из видео. 1 означает извлечение каждого кадра видео. Временные метки для субтитров будут наиболее точными, и сами субтитры могут быть точнее. 2 означает извлечение каждого второго кадра. Временные метки будут слегка смещены, но весь процесс извлечения субтитров значительно ускорится, так как будет использована только половина кадров (общее количество кадров / 2).

**Frame Extraction Batch Size (Размер пакета для извлечения кадров)**: Количество кадров, обрабатываемых каждым ядром ЦП.

## Извлечение текста

Все параметры, связанные с извлечением текста, сгруппированы здесь.

### Извлечение

<img src="images/text%20extract.png" width="400">

**Text Extraction Batch Size (Размер пакета для извлечения текста)**: Количество кадров, обрабатываемых каждым ядром/процессом ЦП или ГП.

**Text Drop Score (Минимальный порог уверенности для текста)**: Минимально допустимый показатель уверенности для каждого извлеченного фрагмента текста. Низкое значение приведет к появлению большего количества неточного текста в сгенерированных субтитрах. Очень высокое значение даст более точный текст, но многие фрагменты будут пропущены.

**Use Line Break (Использовать разрыв строки)**: Разрыв строки будет использоваться, если в одном кадре обнаружено более одной строки текста. Может работать некорректно, если OCR обнаруживает много пробелов в одной строке текста.

### Движок OCR

<img src="images/ocr%20engine.png" width="400">

**OCR Recognition Language (Язык распознавания OCR)**: Язык субтитров в видео, которые необходимо извлечь.

**Use Text Line Orientation Model (Использовать модель определения ориентации строки текста)**: Включение этой опции позволяет использовать модель для определения ориентации текста и его поворота при необходимости перед передачей в модель распознавания. Следует использовать, если какой-либо текст в видео расположен не горизонтально.

**Use Mobile Model (Использовать мобильную модель)**: Определяет, будет ли использована серверная или мобильная версия модели. Серверная модель работает медленнее, но обладает более высокой точностью, тогда как мобильная модель быстрее, но менее точна.

### Производительность OCR

<img src="images/ocr%20perf.png" width="400">

**CPU OCR Processes (Процессы OCR на ЦП)**: Количество ядер ЦП, используемых для извлечения текста из видео. Слишком малое или слишком большое количество увеличит время извлечения.

**CPU Onnx Intra Threads (Внутренние потоки Onnx для ЦП)**: Количество потоков, используемых Onnx для распараллеливания выполнения внутри узлов на ЦП.

**GPU OCR Processes (Процессы OCR на ГП)**: Количество процессов на ГП, используемых для извлечения текста из видео. Используется только при наличии ГП. Использование слишком большого количества приведет к снижению производительности из-за чрезмерной нагрузки.

**_Примечание:_** Для параметров "Процессы OCR на ЦП/ГП" и "Внутренние потоки Onnx для ЦП" может потребоваться тестирование с разными значениями, чтобы определить оптимальные, обеспечивающие высокую загрузку ЦП или ГП.

**Use GPU if available (Использовать ГП, если доступен)**: ГП будет использоваться для ускорения извлечения текста при наличии.

**Auto Optimize Performance (Автоматическая оптимизация производительности)**: Количество процессов OCR и потоков Onnx будет автоматически скорректировано для обеспечения оптимального использования ЦП.

### Генератор субтитров

<img src="images/sub%20gen.png" width="400">

**Text Similarity Threshold (Порог схожести текста)**: Порог для объединения схожих текстов, которые появляются последовательно, но могут немного отличаться из-за погрешностей OCR. Более низкий порог приведет к объединению текстов, которые на самом деле различаются.

**Minimum Consecutive Sub Duration (Минимальная длительность последовательных субтитров)**: Минимальная длительность для субтитров, появляющихся последовательно. Последовательно малая длительность обычно означает, что был обнаружен какой-либо фоновый текст в видео, который появляется на короткое время.

**Max Consecutive Short Durations (Максимальное количество последовательных коротких субтитров)**: Максимально допустимое количество последовательных субтитров с короткой длительностью. Если обнаруженное количество превышает это значение, они будут удалены.

**Minimum Sub Duration (Минимальная длительность субтитра)**: Любой субтитр с длительностью меньше или равной этому значению будет удален.

### Уведомления

<img src="images/notification.png" width="400">

**Notification Sound (Звук уведомления)**: Звук, который будет воспроизведен после завершения обнаружения или генерации субтитров.

**Loop Notification Sound (Зациклить звук уведомления)**: Звук уведомления будет повторяться.
